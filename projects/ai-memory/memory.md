## What It Is
The markdown-only memory system for Claude Code — no vector DB, no daemons, no MCP server, just files + hooks + macOS-`bash`-3.2 shell scripts. This is a **meta-project**: the project being tracked *is* the memory tooling. Single owner (Seyi). Authoritative spec is the repo `README.md`.

Three layers (Karpathy LLM-Wiki pattern):
- **Schema** — `identity.md` (hard rules, injected once/session), `~/.claude/CLAUDE.md`. Outranks everything.
- **Wiki** — `domain/*.md` (cross-project) + `projects/*/memory.md` (engagement-specific). Durable.
- **Scratchpad** — `projects/*/working.md`, injected every prompt while non-empty; matures into the wiki via `/promote-memory`.

Moving parts: one `UserPromptSubmit` hook (`~/.claude/hooks/inject_memory.sh`) emits `<memory:*>` blocks; slash commands in `~/.claude/commands/`; scripts in `scripts/` (`_lib.sh`, `regenerate-index.sh`, `lint-memory.sh`, `archive-cleanup.sh`, `new-project.sh`, `memory-pin.sh`, `link-skills.sh`, `executor.sh`, `codex-mem.sh`, the `taskprovider/` package + `taskctl`) with dependency-free bash-3.2 tests under `scripts/tests/`.

## Architecture Decisions
- **Markdown over a DB** — every editor/diff/grep works; zero infra; git-trackable.
- **Hook-injected, not retrieved** — context arrives in-band. Full payload (identity → project memory → index → working) is injected at SessionStart, on `@memory`, and on the first prompt after compaction; every other prompt gets only the lightweight `<memory:active>` breadcrumb. Domain files are the exception: lazy-read on index trigger-match, never auto-injected.
- **Frontmatter-driven catalog** — `index.md` AUTOGEN block is regenerated from frontmatter (`/reindex`), so it can't lag the files.
- **Lean index** — `index.md` is a path-less roster; paths are derivable (`projects/<name>/memory.md`, `domain/<topic>.md`), metadata (`tags`/`repo`/`repo_path`, domain `triggers`) stays in the source files. Codex's `AGENTS.md` keeps absolute paths (its shell tool reads by path).
- **Per-session injection markers** (`~/.claude/memory_sessions/<session_id>`, self-expire after 2 days) — once-per-session blocks don't re-inject; concurrent sessions don't collide.
- **Compaction recovery** — `PreCompact` stdout can't inject; the reliable channel is `UserPromptSubmit`. `SessionStart source=compact` writes a per-session sentinel; the next prompt re-injects the full payload once and clears it.
- **Project detection per-prompt** — the hook reads `cwd`, walks up for a `.claude/memory-project` marker; no marker → no project (memory dormant, no global fallback, so concurrent repos never collide). `.active_project` machinery still exists but is left empty by deliberate decision.
- **Bidirectional repo↔project map** — forward marker `.claude/memory-project` + reverse `repo`/`repo_path` frontmatter, both written by `/pin` (`memory-pin.sh`). `resolve_repo_path` resolves a checkout per-environment: `repo_path` relative to `projects_root()` (`AI_MEMORY_PROJECTS_ROOT`, default `$HOME/Projects`), git-remote `repo` as portable fallback. `lint-memory.sh` catches reverse-map drift. **`repo_path` is stored relative, never absolute** — the env-var root supplies the prefix so paths resolve identically across host and sandbox. **Self-referential exception:** the `ai-memory` meta-project's checkout *is* the memory tree, so its `repo_path` is the sentinel `$MEMORY_DIR` (and `$MEMORY_DIR/<sub>` is supported) — `resolve_repo_path` and `lint-memory.sh` resolve it against `MEMORY_DIR` rather than `projects_root()`, keeping the committed value machine-independent (each machine sets `MEMORY_DIR` to wherever its tree lives).
- **Distributed cross-project relationships** — a relationship lives in the project where the work *starts* (`## Related Projects`), never in an umbrella. Siblings are **delegated to subagents, never loaded** into the orchestrator thread; plan-set execution pauses at human/CI gates.
- **Orchestrator / Executor / Validator workflow** — Claude main session = orchestrator; an executor runs delegated work; a Claude `Agent` (`sonnet`) validator checks output against the plan's `## Success criteria`. Harness `TaskCreate`/`TaskUpdate` are banned (enforced by `block_task_tools.sh`); `todo.md` is the single source of truth. Archive is never read unless asked. Executors are blocked from any apply/merge/destructive action on running infrastructure.
- **User-selectable executor** — the executor is not hardcoded to Codex. `scripts/executor.sh` resolves `AI_MEMORY_EXECUTOR` (from `config.local.sh`; default `claude-subagent`) and routes by *plane*: `--which` → `subagent` (Agent tool) or `cli:<key>` (run `--run`, which execs the CLI or bounces back to the subagent). Built-in types: `claude-subagent` and `codex` (via `codex-mem.sh --executor`); generic CLIs via `AI_MEMORY_EXECUTOR_CMD_<key>` (`{prompt}` substituted); missing binary auto-falls-back. **Which executor a given instance uses is config-driven, never recorded here** — set `AI_MEMORY_EXECUTOR` in the gitignored `config.local.sh`; the orchestrator discovers it at delegation time by running `executor.sh --which`, so this tracked doc stays instance-agnostic. The codex deny-rules file (`~/.codex/rules/default.rules`) is optional codex-only hardening; the no-merge deny-list is restated in every delegation prompt regardless of executor.
- **Task-provider layer + `/task`/`/start`** — pluggable capture/track/execute subsystem; stdlib-only Python `scripts/taskprovider/` (generic registry: env value `MEMORY_TASK_PROVIDER` *is* the provider module → no factory edit to add a backend), `providers/local.py` (flat `tasks/<slug>.md`) + `providers/notion.py` (urllib-only). Backend is a projection + intake queue, push-dominant, never bidirectional. `scripts/taskctl` bash wrapper; `/task` (add|list|done|archive|show), `/start <ref>` (runs the brainstorm gate → scaffolds a plan into the task's own project).
- **Brainstorming skill (gated Tier-3 design pass)** — runs the collaborative design pass only for Tier-3 feature tasks with open design questions; silent on Tier 1/2 and settled Tier-3. Terminal handoff is `/new-plan`, folding the approved design into the plan's Goal/Success criteria/Design/Risks. Orchestrator-only, no separate committed spec.
- **Task Contract — success criteria** — plan-tier tasks carry explicit checkable success criteria (the Validator's contract). Policy in `identity.md` → `### Task Contract`; enforcement is template-only (`/new-plan` scaffold). A criteria-less plan is a process failure (draft them first), not a license to invent a bar.
- **Multi-git-provider support** — projects span GitHub, Bitbucket Cloud, Azure DevOps. Provider is inferred from each project's `repo` remote host (no `git_provider` field). PR/repo tooling routes by provider: `gh` / `bkt` skill / `az repos`. The no-merge guardrail extends to all three (Azure merge = `az repos pr update --status completed`).
- **`renovate-manager` review skill** — one dispatcher skill routing by Renovate PR manager type to per-domain reference reviewers (helm vs terraform). Runs as parallel read-only subagents (Analyzer + Upstream) whose context discipline is the point: the orchestrator loads only the small body + changed-file paths; heavy work (diff, clone, `validate`, release-notes) runs inside the subagents. Review memory lives inside the skill dir (gitignored), cache-gated. Project resolved from the PR repo URL via the reverse-map — if it doesn't resolve, ask, never assume.
- **Skills owned by memory, symlinked into harnesses** — canonical store `skills/<name>/` under the memory root; each runtime's skills dir holds per-skill symlinks back into it (one source of truth, harness-agnostic). Sync via `scripts/link-skills.sh`.
- **Skill subsystem — declared, enforced write boundary + self-rating.** Every skill carries `metadata.tier` (`target-read-only` | `target-write`), a neutral label enforced harness-agnostically — *not* Claude's `allowed-tools`. Three write zones: the target tree (tier-gated) · the skill's own `skills/<name>/` folder (always writable, no declaration) · everything else off-limits. Enforcement is layered and *detective*: execpolicy (destructive floor) + a post-run git-diff boundary check (in-session PostToolUse/Stop hooks; Codex executor trigger deferred) + the Validator (correctness). `validate-skills.sh` statically checks the schema; `new-skill.sh` / `install-skill.sh` author / intake to it. First-party workflow skills also carry an **on-request** self-rating block (skill friction, a separate axis from the Validator); loop membership is **marker-derived**, not a static list. Authoring rule: every script action keeps a hand-editable equivalent (Two-Path principle). Shipped in PR #8; full design + decision record in `archive/plans/skill-subsystem.md`.
- **Knowledge lifecycle** — per-project `working.md` → `domain/<topic>.md` or project `memory.md` Decisions Log (via `/promote-memory`) → packaged skill (manual graduation when a domain file grows past ~50 lines or holds a coherent reusable pattern).
- **This meta-project is tracked in-repo** — unlike every other project (external repo + client data → gitignored), `projects/ai-memory` documents *this* repo and carries no secrets, so its durable artifacts are version-controlled via a `.gitignore` carve-out: **tracked** = `memory.md`, `plans/`, `todo.md`, **and `archive/`** (past work stays visible in git — the memory system simply never *loads* `archive/`, a runtime convention, not a tracking one); **ignored** = only `working.md` (per-machine live scratch). Repo is private, so internal repo/org names are acceptable to track. Consequence: routine `memory.md`/`todo.md` edits and plan-archival become git changes — **commit these directly to `main` (no PR)**; reserve a branch + PR for new features / substantive code changes.
- **Per-environment config loader; instance config never in tracked memory** — `_lib.sh` and `taskctl` source a gitignored `config.local.sh` next to the tree; ship `config.local.sh.example`. Because the same core/memory is shared across multiple instances, **machine/instance-specific choices live in `config.local.sh`, not in this tracked `memory.md`** — projects root (`AI_MEMORY_PROJECTS_ROOT`), executor (`AI_MEMORY_EXECUTOR`), task provider (`MEMORY_TASK_PROVIDER`), etc. Scripts read them by sourcing `_lib.sh`; the orchestrator reads runtime-derived ones via their tool (e.g. `executor.sh --which`). The committed default root is `$HOME/Projects`.
- **Codex adapter** — `codex-mem.sh` regenerates `~/.codex/AGENTS.md` on every launch from the same memory files (no hooks in Codex; only injection mechanism differs). `--executor` = `exec --sandbox workspace-write` network-on; `--executor-bare` strips the AGENTS.md merge (~13k tokens) for read-only review subagents. Mid-session memory edits don't appear until relaunch.
- **Non-goals:** no `PreToolUse` deny enforcing `todo.md` (documented convention only); no bootstrap script (README is the source of truth for rebuilding).

## Known Constraints / Gotchas
- **This repo IS under git, but personal content is `.gitignore`d.** Authoritative list is `.gitignore` — verify there. Tracked (distributable source): `scripts/`, `claude/`, `agents/`, generic `skills/`, `install.sh`, `README.md`, `identity.md`, `config.local.sh.example`, `projects/_template`, `domain/_template.md`. Ignored (personal/runtime): `.claude/`, `.sessions/`, `/index.md`, `/domain/*` (except `_template.md`), `/projects/*` (except `_template` **and `ai-memory`**), `/tasks/`, `/archive/`, `/config.local.sh`, `/skills/fiter-infrastructure-analyzer/`, `/docs/superpowers/`. Live per-engagement memory never gets committed **except this `ai-memory` meta-project**. Never commit secrets.
- **Scripts target macOS `bash` 3.2** — no `mapfile`, no associative arrays. Resolve the tree via `MEMORY_DIR`. `_lib.sh` self-locates `MEMORY_DIR` via `BASH_SOURCE` (correct under bash, empty under the Bash-tool's interactive zsh — don't sanity-check `. scripts/_lib.sh` from zsh).
- **`archive/` is never read unless the user explicitly asks** — and never deleted during a "reorganize memory" pass (audit trail).
- **Frontmatter is the contract** — missing/malformed fields break the index; `lint-memory.sh` catches it. `_template/` is excluded from index/lint/regeneration.
- **Working memory is per-project and injected every prompt** — keep it short or it bloats every request and busts the cache prefix. Two terminals on the same project both writing `working.md`: last write wins (no merge; acceptable for solo use).
- Slash commands are indexed at session start — a new command needs a session restart to autocomplete.
- Oversized hook stdout is routed to a `<persisted-output>` file with a 2KB preview (the hard 10k cap is gone), but a bloated `working.md` still re-ships every prompt.
- **Codex execpolicy** `decision` keyword is `"forbidden"` (not `deny`/`reject`/`block`); invalid values error at `codex exec` time, not parse time. Executor invocation needs `--skip-git-repo-check` when the workdir isn't a git repo.
- **Notion provider:** the API can add/reorder but cannot rename/delete select/status options (rename = add + reorder + leave old vestigial). For `/task`/`/start` to route to Notion, creds + `MEMORY_TASK_PROVIDER=notion` + `NOTION_STATUS_KIND` must be in `~/.zshenv` (NOT `.zshrc`) — the Bash tool's non-interactive zsh sources `.zshenv`.
- **Plan-mode plan path:** the harness default (`~/.claude/plans/<slug>.md`) conflicts with the canonical path (`projects/<active>/plans/<name>.md`). In plan mode, always write to the memory-system path.
